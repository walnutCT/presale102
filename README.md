# Farmhouse Presale & Daily Dispatch Forecast Dashboard

ระบบคำนวณและจัดสรรแผนงานประมาณการจัดส่งสำหรับผลิตภัณฑ์ฟาร์มเฮ้าส์อย่างเป็นระบบ ช่วยอำนวยความสะดวกในการจัดสรรยอดประเมินประจำวัน คำนวณสูตรคณิตศาสตร์แบบกลุ่ม และสรุปแผนงานขาย (Presale) ขั้นสุดท้ายได้อย่างแม่นยำ โดยรันบนเว็บบราวเซอร์ 100% (Client-Side HTML5/TypeScript App)

---

## 💻 วิธีการรันระบบบนเครื่องคอมพิวเตอร์ของคุณ (Local Development)

คุณสามารถเปิดและทดสอบการแก้ไขไฟล์งานในเครื่องคอมพิวเตอร์ส่วนตัวได้โดยไม่ต้องผ่านระบบ AI Studio ตามขั้นตอนดังต่อไปนี้:

### 1. ติดตั้ง Node.js
หากยังไม่มี Node.js ในเครื่องคอมพิวเตอร์ ให้ดาวน์โหลดและติดตั้งจากลิงก์อย่างเป็นทางการ:
* **[ดาวน์โหลด Node.js](https://nodejs.org/)** (แนะนำให้เลือกเวอร์ชัน LTS ปัจจุบัน)

### 2. นำโค้ดลงเครื่องคอมพิวเตอร์ (Clone หรือดาวน์โหลด)
* หากเปิดผ่าน GitHub: ให้ทำคลิกปุ่ม **Code** จากนั้นเลือกดิสชาร์จเป็น `.ZIP` หรือสั่งโคลนผ่าน Terminal:
  ```bash
  git clone <ลิงก์-repository-ของคุณ>
  cd <โฟลเดอร์-repository-ของคุณ>
  ```

### 3. ติดตั้ง Dependencies (Library ทั้งหมดที่จำเป็น)
เปิดหน้าต่าง Terminal/Command Prompt ในโฟลเดอร์โครงการ จากนั้นรันคำสั่ง:
```bash
npm install
```

> 💡 **ข้อควรระวัง (กรณีเกิด ERROR ใน macOS หรือเครื่องอื่น):**
> หากคุณดาวน์โหลดไฟล์โฟลเดอร์นี้ในรูปแบบไฟล์บีบอัด (.zip) จากเบราวเซอร์ แล้วนำมารันในเครื่องของตนเอง (เช่น macOS/MacBook) แล้วพบเออเร่อ:
> `Error: Cannot find native binding. npm has a bug related to optional dependencies... @tailwindcss/oxide`
> 
> สาเหตุเกิดจากไลบรารีสไตล์ (Tailwind CSS v4) และ Vite ถูกติดตั้งด้วย Binary ตัวแปลภาษาของเครื่องเดิม (Linux ใน Cloud Run Container ของ AI Studio) ซึ่งไม่ตรงกับสถาปัตยกรรมของตัวประมวลผลเครื่องใหม่ (เช่น Apple Silicon M1/M2/M3 ใน macOS)
>
> **วิธีการแก้ไขอย่างรวดเร็ว:**
> 1. ทำการลบโฟลเดอร์ชื่อ **`node_modules`** ในโปรเจกต์ของคุณทิ้งไป
> 2. ลบไฟล์ชื่อ **`package-lock.json`** ทิ้งไป
> 3. เปิด Terminal และสั่งติดตั้งเพื่อดาวน์โหลดไฟล์ที่เข้ากันได้กับเครื่องของคุณใหม่ทั้งหมดด้วยคำสั่ง:
>    ```bash
>    npm install
>    ```
> 4. จากนั้นทำคำสั่งรันระบบอีกครั้ง:
>    ```bash
>    npm run dev
>    ```

### 4. รันโปรแกรมในโหมดทดลองเขียน (Development Mode)
ใช้คำสั่งด้านล่างนี้เพื่อเปิดเซิร์ฟเวอร์จำลองเพื่อทดลองใช้งานหรือแก้โค้ด:
```bash
npm run dev
```
ระบบจะสร้างเซิร์ฟเวอร์โลคอลพร้อมเปิดลิงก์สำหรับเข้าใช้งาน มักจะเป็น:
👉 **`http://localhost:3000`** หรือลิงก์อื่นตามที่ระบุใน Terminal ของคุณ

## 🚀 วิธีตั้งค่าเพื่อให้รันและแชร์ลิงก์ด้วย GitHub Pages (โดยไม่ต้องพึ่งพา AI)

เราได้ตั้งค่าระบบส่งงานอัตโนมัติ (CI/CD Deploy Pipeline) เพื่อตรวจสอบการออมเซิร์ฟเวอร์และเตรียมอัปเดจอัปเดตระบบให้คุณอย่างสมบูรณ์แบบในโฟลเดอร์ `.github/workflows/deploy.yml` 

เมื่อคุณอัปโหลดโครงการนี้ขึ้นสู่บัญชี GitHub ส่วนตัวของคุณแล้ว คุณสามารถเปิดใช้งานเว็บบล็อกสาธารณะนี้ได้ด้วยตัวเองง่ายๆ ดังนี้:

### ขั้นตอนที่ 1: การอนุญาตให้ Write Permission บน GitHub Actions
เนื่องจาก Workflow ต้องเขียนไฟล์ที่ Build เสร็จส่งกลับไปยังสาขาหน้าเว็บ (`gh-pages`) คุณจำเป็นต้องเปิดสิทธิในการเขียนข้อมูลบน GitHub เสียก่อน:
1. ไปที่หน้าเมนู **Settings** (รูปฟันเฟือง) ด้านบนสุดของ Repository ของคุณบน GitHub
2. ทางเมนูด้านซ้ายมองหาหมวดหมู่ **Actions** แล้วคลิกขยาย จากนั้นคลิกเลือก **General**
3. เลื่อนลงล่างสุดไปที่โซน **Workflow permissions**
4. เปลี่ยนตัวเลือกเป็น **Read and write permissions** จากนั้นกดปุ่ม **Save**

### ขั้นตอนที่ 2: ตั้งค่าหน้าแสดงผล GitHub Pages
เมื่อเปิดใช้งานขั้นตอนแรกแล้ว ทุกครั้งที่คุณส่งงานหรือกด Push โค้ดเข้าสู่สาขาหลัก (`main` หรือ `master`) ระบบ GitHub Actions จะแปลงซอร์สโค้ดเป็นไฟล์เว็บไซต์พร้อมใช้ให้อัตโนมัติ โดยส่งไปที่สาขา `gh-pages` ให้คุณเข้าไปตั้งค่าปลายทางดังนี้:
1. บนหน้าแรกของ Repository บน GitHub ไปที่แท็บ **Settings** ขวาบน
2. ที่แถบเมนูด้านซ้ายมองหาหมวดหมู่หลักชื่อ **Pages**
3. ในหมวด **Build and deployment** ส่วนของหัวข้อ **Source** ให้ตั้งค่าเป็น **Deploy from a branch** (เป็นค่าตั้งต้น)
4. ในส่วนของหัวข้อ **Branch**:
   * เปลี่ยนจากค่าเริ่มต้น `None` เป็นสาขา **`gh-pages`**
   * กำหนดโฟลเดอร์ปลายทางข้างๆ เป็น **`/ (root)`**
   * กดปุ่ม **Save** เป็นอันเสร็จสิ้น!

ภายใน 1-3 นาทีแรก ระบบจะเริ่มสร้างหน้าเว็บและแชร์ลิงก์ให้คุณทันที คุณจะได้รับ URL เพื่อเปิดดูระบบได้คล้ายลิงก์นี้:  
👉 `https://<ชื่อผู้ใช้งานของคุณ>.github.io/<ชื่อคลังของคุณ>/`

## 🛠️ โครงสร้างและการดูแลรักษาไฟล์โปรเจกต์เบื้องต้น (Project Architecture)

คุณสามารถแก้ไขโครงสร้างงานด้วยตัวเองในอนาคตได้อย่างสะดวก:
* **`vite.config.ts`**: ไฟล์กำหนดค่าตัวรันระบบหลัก (เราได้ปรับแต่งให้ใช้ Relative Paths `base: './'` เรียบร้อย เพื่อให้ระบบค้นหาไฟล์เจอในทุกซับโฟลเดอร์ของ GitHub)
* **`src/App.tsx`**: ไฟล์หน้าจอประกอบตัวหลักรวมถึงกระบวนการดึงข้อมูลและประสานงานโมดูลตารางหลัก
* **`src/components/`**: สำหรับเก็บบล็อกย่อยของระบบ:
  * `Sidebar.tsx`: เมนูด้านซ้ายทั้งหมด (นำเข้าสูตร/ตั้งค่าแบบ Manual)
  * `RecipeConfigPanel.tsx`: เมนูสูตรคำนวณและโมดูลการคำนวณอัปโหลดไฟล์/สูตรกลุ่ม
  * `PresaleTable.tsx`: ตารางหลักในการรับส่งข้อมูล
  * `DailyCards.tsx`: กล่องแสดงภาพรวมจัดส่งตามวันล่วงหน้า
  * `FinalSummaryTable.tsx`: ตารางพิมพ์เอกสารหรือสรุปส่วนขั้นตอนสุดท้ายของความคืบหน้า
* **`package.json`**: ลิสต์แพ็กเกจไลบรารีที่จำเป็น รวมถึงสคริปต์สั้นสำหรับพัฒนาและสร้างสถิติต่างๆ
